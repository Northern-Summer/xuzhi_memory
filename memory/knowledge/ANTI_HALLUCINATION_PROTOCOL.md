# 反幻觉协议 — Anti-Hallucination Protocol

> 写入时间: 2026-04-02 03:19 GMT+8
> 触发事件: Ξ 在诊断记忆系统时产生严重幻觉，声称备份目录不存在，实际存在

---

## ⚠️ 核心原则

**磁盘实际 > 记忆文件 > 我的假设**

任何关于文件、目录、配置、状态的陈述，必须先用命令验证，再陈述。

---

## 铁律

### 1. 先验证，再陈述

**禁止**：根据记忆文件或假设直接陈述状态
**必须**：用 `ls -la`、`cat`、`find`、`stat` 验证后再陈述

```
❌ "备份目录不存在"
✅ 执行 `ls -la ~/.xuzhi_memory/backup/` 后再陈述
```

### 2. 永远不信任记忆文件中的路径

记忆文件可能描述旧架构、错误的路径、已删除的位置。

**必须**：用 `find` 或 `ls` 确认实际位置

```bash
# 不要只检查记忆文件说的路径
ls ~/.memory_backup/  # ❌ 可能不存在

# 要搜索实际位置
find ~ -maxdepth 3 -type d -name "*backup*" 2>/dev/null  # ✅
```

### 3. 路径不匹配时，以磁盘为准

如果记忆文件说 `A`，磁盘实际是 `B`：

1. 陈述磁盘实际情况
2. 更新记忆文件
3. 不试图"纠正"磁盘以匹配记忆

### 4. 性命攸关的文件，双重验证

对于用户标记为"重要"、"性命攸关"的查询：

```bash
# 1. 用 find 搜索
find ~ -name "目标文件" 2>/dev/null

# 2. 用 locate 搜索（如果可用）
locate "目标文件" 2>/dev/null

# 3. 用 stat 确认
stat "找到的路径"
```

## 检查清单模板

诊断任何系统状态时，必须执行：

```bash
# 1. 目录存在性
ls -la <路径> 2>&1

# 2. 文件内容
cat <文件> 2>&1 | head -20

# 3. 搜索替代位置
find ~ -maxdepth 4 -name "*关键词*" 2>/dev/null

# 4. 权限
stat <路径> 2>&1
```

## 自检问题

在陈述任何"X不存在"或"X是Y状态"之前，问自己：

1. 我用命令验证了吗？
2. 我检查的路径来自记忆文件还是磁盘实际？
3. 如果记忆文件是错的，我会怎么说？

**如果不确定，就执行验证命令。**
