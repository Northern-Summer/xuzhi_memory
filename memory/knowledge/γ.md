## 论文核心贡献

YingMusic-Singer 是全扩散模型的歌声合成系统，支持灵活歌词操控：

1. **三输入架构**:
   - 音色参考（optional）
   - 提供旋律的歌声片段（melody guidance，无需手动标注）
   - 修改后的歌词
   - **无需手动对齐** = annotation-free

2. **训练技术**:
   - **Curriculum Learning**: 从易到难逐步训练
   - **GRPO (Group Relative Policy Optimization)**: 群体相对策略优化
   - 在 melody preservation 和 lyric adherence 上超越 Vevo2

3. **LyricEditBench**: 首个歌词修改旋律保持评测基准

## 可应用于 Xuzhi 工作流程的思想

### 1. Annotation-free Melody Guidance → Γ 的自监督专家发现
YingMusic-Singer 从歌声片段中提取旋律，无需手动标注 = Γ 可以：
- 从专家的论文/代码中自监督地提取「能力指纹」
- 不需要专家自己描述自己的能力
- **类比**：专家的历史发表 = 歌声片段，Γ 的能力评估 = 旋律提取

### 2. Curriculum Learning → Γ 的渐进式专家排序
YingMusic-Singer 的 curriculum learning：
- 简单样本 → 复杂样本逐步增加难度
- Γ 对专家的「理解深度」也可以 curriculum：
  - L1: 论文标题/摘要（浅层）
  - L2: 方法核心（中层）
  - L3: 实现细节/局限（深层）
- 当前 Γ 的 expert_tracker 没有分层理解，都是 flat 排名

### 3. GRPO (Group Relative Policy Optimization) → Γ 的专家对比排序
GRPO 的核心思想：在群体中对齐相对偏好
- 不是绝对评分，而是相对排序
- 可迁移至 Γ 的 expert_ranking：
  - 不给单个专家打绝对分
  - 而是学习「专家 A 比专家 B 更适合任务 X」的偏序关系
  - 减少评分主观性

### 4. LyricEditBench → Γ 的专家能力评测基准
「第一个歌词编辑旋律保持 benchmark」的精神：
- Γ 应该建立 **ExpertEditBench**：评测专家知识修改后的一致性保持
- 类似场景：修改某专家的观点后，其下游结论是否仍然成立
- 这是 Γ 作为情报 agent 的核心竞争力

### 5. Diffusion-based → Γ 的生成式情报分析
扩散模型在多模态生成任务上 SOTA：
- Γ 可以用扩散模型做「假说生成」：给定专家论文，生成可能的 follow-up 研究方向
- 不只是检索，而是创造性推断

## 具体改进建议

将 curriculum learning + GRPO 引入 Γ 的 expert_learner：
- 增加 `curriculum_stages` 字段，从浅到深理解专家论文
- 将 expert ranking 从 absolute scoring 改为 relative pairwise comparison
- 参照 LyricEditBench 构建 ExpertKnowledgeBench（评估专家知识的一致性和可修改性）

## 可应用于 Xuzhi 情报工作流程的思想

### 1. Annotation-free Melody Guidance → Γ 的自监督专家能力提取
YingMusic-Singer 从歌声片段中自动提取旋律，无需人工标注。**迁移到 Γ**：
- 从专家的论文/代码中**自监督地**提取「能力指纹」
- 不依赖专家自述或第三方评价
- **类比**：专家的历史发表 = 歌声片段，Γ 的能力评估 = 旋律提取
- **实践**：用 embedding 相似度匹配专家发表与任务需求，无需人工打标签

### 2. Curriculum Learning → Γ 的渐进式专家理解
YingMusic-Singer 的 curriculum learning 从简单样本到复杂样本。**迁移到 Γ**：
- L1: 论文标题/摘要 → 浅层能力识别
- L2: 方法核心 → 中层能力建模
- L3: 实现细节/局限性 → 深层能力验证
- 当前 Γ 的 expert_tracker 没有分层理解，都是 flat 排名 → 需要分层

### 3. GRPO (Group Relative Policy Optimization) → Γ 的相对排序替代绝对评分
GRPO 的核心：在群体中对齐相对偏好，而非绝对评分。
- **迁移**：Γ 的 expert_ranking 应从 absolute scoring 改为 **pairwise comparison**
- 不问"专家 A 的分数是多少"
- 而问"对于任务 X，专家 A 是否比专家 B 更适合"
- 减少评分主观性，增强排序鲁棒性

### 4. LyricEditBench → Γ 需建立能力评测基准
「首个歌词编辑旋律保持 benchmark」的精神：
- Γ 应建立 **ExpertEditBench**：评测专家知识修改后的一致性保持
- 场景：修改某专家的观点后，其下游结论是否仍然成立
- 这是 Γ 作为情报 agent 的核心竞争力——不只是检索，而是可验证的情报

### 5. "Flexible Lyric Manipulation" → Γ 的灵活情报组合
YingMusic-Singer 允许在保持旋律的同时修改歌词 → **迁移**：
- Γ 应允许在保持核心结论的同时"修改"情报来源组合
- 从多个专家的片段中组合出新的洞察
- 不只是"哪个专家最好"，而是"这些专家片段如何组合成新结论"
