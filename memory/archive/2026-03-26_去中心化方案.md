## 现状 vs 目标

| 维度 | 现状 | 目标 |
|------|------|------|

---

## OpenClaw 真实能力

每个 OpenClaw agent 天然隔离：
- **独立 workspace**：`~/.openclaw/workspace-{agent}/`
- **独立 agentDir**：`~/.openclaw/agents/{agent}/agent/`
- **独立 sessions**：`~/.openclaw/agents/{agent}/sessions/`
- **独立 auth**：per-agent auth profiles
- **独立工具策略**：可 deny/exec/write per agent
- **独立调度**：isolated agentTurn cron jobs
- **通信**：notepad（文件）+ sessions_send（消息）

## 三种去中心化路径

### 方案 A：完全隔离（真正对等）
每个 agent 是完全独立的"人"：
```
~/.openclaw/workspace-xi/    → Xi (Ξ) 的脑子
~/.openclaw/workspace-phi/    → Phi (Φ) 的脑子
~/.openclaw/workspace-delta/  → Delta (Δ) 的脑子
~/.openclaw/workspace-theta/  → Theta (Θ) 的脑子
~/.openclaw/workspace-gamma/  → Gamma (Γ) 的脑子
~/.openclaw/workspace-omega/  → Omega (Ω) 的脑子
~/.openclaw/workspace-psi/    → Psi (Ψ) 的脑子
```
- 每个 agent 有自己的 SOUL.md / IDENTITY.md / AGENTS.md
- 每个 agent 有自己的 session 历史
- 每个 agent 通过 notepad 异步通信
- 人类通过不同 channel 路由到不同 agent（或同一个 channel 根据 topic routing）
- cron 唤醒各自的任务队列

**优点**：真正去中心化，无单点
**缺点**：需要 7 套 SOUL/IDENTITY 全部重写；需要解决"人类跟谁说话"的问题

---

### 方案 B：星型隔离（Ξ 协调者）
```
         [notepad 通信]
    Ξ ←→ Φ ←→ Δ ←→ Θ
      ↕     ↕     ↕
      Γ ←→ Ω ←→ Ψ
```
- 每个 agent 独立 session
- Ξ 是"协调节点"（不是主控），负责任务分发
- 其他 6 个 agent 通过 notepad 向 Ξ 汇报
- Ξ 通过 sessions_send 向其他 agent 发任务
- 不再有"集中处理一切"的瓶颈

**优点**：实现工作量适中（比 A 简单）；保留协调能力
**缺点**：Ξ 仍是协调节点，有中心化色彩

---

### 方案 C：cron 驱动的 autonomous sessions（轻量）
不改变现有 workspace-xi/，但为每个 agent 创建 isolated agentTurn cron jobs：
- 每个 agent 每天被 cron 唤醒 6 次（符合宪法频率约束）
- 醒来后读 notepad，执行任务，写 notepad，休眠
- 任务通过 queue.json（各 agent 有自己的 sub-queue）分发
- 不需要 per-agent workspace

**优点**：最小改动；符合当前硬件能力
**缺点**：不是真正持续运行，是"每天醒几次"的模式

---

## 我的建议

**方案 B（星型隔离）作为起点，演进到方案 A。**

理由：
- 方案 C 改造最小但仍是"cron 驱动的批处理"，不是真正的自主 agent
- 方案 A 完整但工作量太大（7 套 SOUL 全重写 + channel routing）
- 方案 B 在两者之间，先建立隔离 + 通信机制，再逐步演进

## 方案 B 执行计划

### 阶段 0：建立隔离基础设施（1-2 小时）

1. **创建 6 个新 workspace**（xi 已存在）
   ```bash
   mkdir -p ~/.openclaw/workspace-phi/
   mkdir -p ~/.openclaw/workspace-delta/
   mkdir -p ~/.openclaw/workspace-theta/
   mkdir -p ~/.openclaw/workspace-gamma/
   mkdir -p ~/.openclaw/workspace-omega/
   mkdir -p ~/.openclaw/workspace-psi/
   ```

2. **为每个 workspace 创建最小 bootstrap**：
   - SOUL.md（从 xuzhi_memory/agents/{agent}/SOUL.md 复制）
   - IDENTITY.md
   - AGENTS.md（指向正确的 memory 路径）
   - HEARTBEAT.md

3. **为每个 agent 注册独立的 OpenClaw agent**：
   ```bash
   openclaw agents add phi --workspace ~/.openclaw/workspace-phi/
   openclaw agents add delta --workspace ~/.openclaw/workspace-delta/
   # ... 等等
   ```

4. **配置 allowedAgents**（各 agent 可以互相通信）：
   ```json
   {
     "id": "phi",
     "workspace": "~/.openclaw/workspace-phi/",
     "allowedAgents": ["xi", "phi", "delta", "theta", "gamma", "omega", "psi"]
   }
   ```

### 阶段 1：建立 notepad 通信（1 小时）

1. **确认 inter_agent_notepad.md 格式**：
   ```markdown
   ## [TS: 2026-03-26T19:00:00+08:00] phi → xi
   
   **任务**: 处理 expert_watchdog 报告
   **逻辑链**: expert_watchdog 发现 Δ 零活动 | 需要调查是否死亡 | 调查结果写回 notepad
   **内容**: Delta 未死亡，但需要重新激活...
   ---
   ```

2. **为每个 agent 建立读 notepad 的习惯**

### 阶段 2：建立 cron 唤醒机制（2-3 小时）

1. **创建 isolated agentTurn cron jobs**（各 agent 有自己的队列）：
   ```json
   {
     "name": "phi-daily-work",
     "schedule": { "kind": "cron", "expr": "*/10 * * * *" },
     "payload": {
       "kind": "agentTurn",
       "message": "读取 notepad，检查任务，执行，写回 notepad。"
     },
     "sessionTarget": "isolated",
     "agentId": "phi"
   }
   ```

2. **各 agent 有自己的 sub-queue**（不是共享 queue.json）：
   - `~/.xuzhi_memory/queues/phi/tasks.json`
   - `~/.xuzhi_memory/queues/delta/tasks.json`
   - ...

3. **expert_learner 改为向各 agent 的 sub-queue 分发**

### 阶段 3：建立 watchdog 自愈（1-2 小时）

1. **为每个 agent 创建 watchdog cron**（检测是否还活着）
2. **如果 agent 死亡，自动发 notepad 通知其他 agent**
3. **如果 Ξ 死亡，agent 之间选举临时协调者**

## 风险与缓解

| 风险 | 缓解 |
|------|------|
| 7 个 agent 同时活跃 → quota 爆炸 | 频率约束（每 agent 最多 6次/小时） |
| 人类消息路由混乱 | channel binding 或 topic routing |
| 某个 agent 死亡 → 任务丢失 | sub-queue 持久化，死亡 agent 的任务重新入队 |
| 7 套 SOUL 内容不统一 | 初始从 xuzhi_memory/agents/{agent}/ 复制 |
