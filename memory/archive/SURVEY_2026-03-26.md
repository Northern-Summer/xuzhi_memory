## 6.2 wakeup_agent.sh（脉冲触发后的实际执行者）

**真实功能（三步链）：**

```
第1步：认领任务
  - 读取 tasks.json 中"等待"任务
  - 优先选择与agent部门匹配的任务
```

## 8.3 Rate Limiter算法

```
capacity（安全容量）计算：
  quota > 60%  → 8（全速）
  quota 40-60% → 4（保守）
  quota 20-40% → 2（谨慎）← 当前24.2%，所以cap=2
  quota < 20%  → 1（极低）

当前quota=24.2%，cap=2，tokens=0。
每次派发消耗1 token，最多2个请求填满窗口。
窗口300秒后重置，但重置后立即又被占满。
```

## 8.4 quota_guard.py的路径问题

rate_limiter.py的get_quota_pct()读取 ~/.openclaw/workspace/quota_status.json（**不存在**），失败后默认返回50%。

但日志显示"quota=24%"，说明某处确实读到了真实值。可能quota_monitor.py在某处更新了rate_limit_state.json中的quota_cache。

# 十七、HEARTBEAT.md修复历史

## 17.1 原始错误格式

之前所有agent的HEARTBEAT.md都是**Python代码格式**（import json, from pathlib等），这是完全错误的。OpenClaw heartbeat机制是：**读取HEARTBEAT.md作为文本检查清单，让agent自己决定做什么，不是执行Python**。

所以之前写的那些Python心跳逻辑（quota检查、任务检查、spawn等）**从未被正确执行过**。

## 17.2 正确格式（22:52修复）

```markdown
# HEARTBEAT.md

## 检查清单
- 确认自己仍在运行
- 检查上次心跳以来是否有新消息或任务
- 有紧急事项需要报告则报告，否则回复 HEARTBEAT_OK
```

这个格式由OpenClaw heartbeat机制读取，agent在session中处理。
