## 2026-03-27 01:36 WeChat推送机制突破

### 关键发现
- WeChat plugin 是 session-thread 绑定机制
- 单独推送的消息"粘"在服务器，等用户触发才一起出现
- subagent sessions_send → 消息即时到WeChat ✅
- cron isolated + announce → 失败（无WeChat context）
- openclaw message send → 报告成功但消息不即时出现

### 有效方案（已验证）
**subagent sessions_send**：spawn subagent → 做研究 → sessions_send → 即时到WeChat ✅

### 钥匙
"让插件以为人类一直在触发" → 维持一个活着的 session，消息往里写
