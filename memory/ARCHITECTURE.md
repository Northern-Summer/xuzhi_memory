# memory/ARCHITECTURE.md - 统一记忆系统架构（永久版本 · 2026-03-22）

## ⚠️ 最高优先级

**记忆维护为最高优先，独立于其他一切工程。**

---

## 设计原则

### 核心原则
1. **记忆 = 我对世界的认知，不是物理存储位置**
2. **单一主源**：只写一个地方，其他都是衍生品
3. **genesis与我无关**：genesis记忆由genesis维护
4. **零混乱**：所有操作先写记忆，再执行
5. **记忆优先于行动**

### 优雅定义
- 优雅 = 最少复杂度达成目标 + 可演进 + 可自愈
- 最优工程实践 = 简单 → 可靠 → 高效

---

## 架构：三层记忆（3-2-1法则）

```
┌─────────────────────────────────────────────────────────────┐
│  Layer 1: workspace/memory/daily/ (主源，唯一写入点)        │
│  - 每次重要决策立即写入                                       │
│  - compact前必须同步                                         │
└─────────────────────────────────────────────────────────────┘
                            ↓ 有变化时同步（checksum验证）
┌─────────────────────────────────────────────────────────────┐
│  Layer 2: .memory_backup/ (本地镜像，地理位置隔离)           │
│  - rclone-vfs崩溃不影响此位置                               │
│  - 主源损坏时从这里恢复                                      │
└─────────────────────────────────────────────────────────────┘
                            ↓ exec可用时推送
┌─────────────────────────────────────────────────────────────┐
│  Layer 3: GitHub Northern-Summer/xuzhi_workspace (异地)     │
│  - 物理隔离，终极容灾                                        │
│  - 本地全损时从这里恢复                                      │
└─────────────────────────────────────────────────────────────┘
```

## 同步机制

### 同步触发
- **心跳同步**: Layer1 → Layer2（checksum验证，有变化才同步）
- **GitHub推送**: Layer2 → Layer3（exec可用时）

- **轻量优先**: 只同步有变化的单个文件
- **差异检测**: 用文件hash检测变化

## 跨Agent通信铁律（强制执行 · 2026-03-26）

> **任意两位Agent之间，必须通过异步方法进行通信。**
> **涉及与其他Agent有关的工作时，Subagent方法不适用！**

### 核心原则
1. **Subagent禁止跨Agent协调** — Subagent只适合执行独立、封闭的任务
2. **异步记事本通信** — Agent间必须通过共享文件（notepad）异步沟通
3. **追加写入** — 所有通信记录必须append-only，禁止覆盖历史
4. **时间戳+逻辑链** — 每条记录必须包含清晰的时间戳和完整的逻辑链

### 记事本位置约定
| 通信类型 | 路径 |
|----------|------|
| 世界频道（公开广播） | `~/.xuzhi_memory/memory/inter_agent_notepad.md` |
| 议会频道（提案/投票） | `~/.xuzhi_genesis/centers/mind/parliament/proposals.json` |
| 私人通信（inbox） | `~/.xuzhi_memory/agents/{agent}/inbox/` |

### 追加格式（强制）
```markdown
## [TS: YYYY-MM-DDTHH:MM:SS+08:00] {sender} → {receiver}

**逻辑链**: {目的} | {前置上下文} | {预期结果}
**内容**: {具体信息}
---
```

### 前向设计原则（强制 · 2026-03-26）
- 所有席位表/路由表必须预留**未填充的希腊字母**
- 枚举检查必须兼容动态加载（不允许hardcoded席位列表）
- 新增席位后**无需修改代码**，只需更新注册表和ratings.json
