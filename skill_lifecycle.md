# 技能生命周期管理

> 基于 AutoSkill (arXiv:2603.01145) 的经验驱动技能演化机制
> 目的：技能创建 → 使用 → 演化 → 废弃的完整生命周期

## 生命周期阶段

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  创建   │ -> │  使用   │ -> │  演化   │ -> │  废弃   │
└─────────┘    └─────────┘    └─────────┘    └─────────┘
     ↑              │              │              │
     └──────────────┴──────────────┴──────────────┘
                     ↓
              经验驱动循环
```

## 1. 创建阶段

**触发条件**：
- 会话中遇到失败（错误、用户纠正、任务未完成）
- 识别到可迁移的行为模式

**创建流程**：
```
失败事件 → 分析原因 → 提取模式 → 写入技能库 → 标记为 v1, g{n}
```

**技能格式**：
```markdown
| "技能名称/描述" | v1 | 来源 | g{n} |
```

**示例**：
```markdown
| "删除前必须先列出、确认" | v1 | 2026-03-30 删除事故 | g1 |
```

## 2. 使用阶段

**触发时机**：
- Session Startup 时读取技能库
- 遇到相关场景时触发技能

**使用记录**：
```yaml
skill: "删除前必须先列出、确认"
used_at: "2026-04-01 16:35"
context: "删除 memory-guardian.disabled 目录"
result: "成功避免误删"
```

## 3. 演化阶段

**触发条件**：
- 技能应用后仍有问题 → 版本升级
- 多次使用发现新边界 → 补充细节
- 与其他技能融合 → 合并

**演化流程**：
```
v1 → v2：技能细节补充
g{n} → g{n+1}：技能库整体升级
```

**示例演化**：
```markdown
v1: "删除前必须先列出、确认"
v2: "删除前必须先列出、确认；涉及系统目录时需要 sudo 密码验证"
```

## 4. 废弃阶段

**触发条件**：
- 技能连续 30 天未使用
- 技能与当前系统架构冲突
- 技能被更好的技能替代

**废弃流程**：
```
标记为 deprecated → 记录废弃原因 → 移到废弃区 → 7天后删除
```

**废弃记录格式**：
```markdown
| "旧技能" | deprecated | 废弃日期 | 废弃原因 |
```
