## HealthFlow 核心机制

### 元规划循环

```
┌─────────────────────────────────────────────────────────┐
│  Meta-Planning Layer（元规划层）                         │
│  ┌───────────────────────────────────────────────────┐  │
│  │ 1. 分析当前任务状态                                │  │
│  │ 2. 选择规划策略                                    │  │
│  │ 3. 生成执行计划                                    │  │
│  │ 4. 监控执行过程                                    │  │
│  │ 5. 评估结果并调整策略                              │  │
│  └───────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  Execution Layer（执行层）                               │
│  ┌───────────────────────────────────────────────────┐  │
│  │ 按计划执行具体任务                                  │  │
│  └───────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
```

## Xuzhi 应用：Expert Tracker 元规划

### 当前状态

Expert Tracker 已经有目标演化机制（SAGA），但缺少"元规划"层。

### 增强设计

**元规划层职责**：

| 阶段 | 动作 | 输出 |
|------|------|------|
| 状态分析 | 评估当前研究进展 | 状态报告 |
| 策略选择 | 选择研究策略 | 策略类型 |
| 计划生成 | 生成具体研究计划 | 行动列表 |
| 执行监控 | 监控研究执行 | 进度报告 |
| 结果评估 | 评估研究结果 | 质量评分 |

**策略类型**：

| 策略 | 适用场景 | 特点 |
|------|---------|------|
| **深度挖掘** | 有价值的问题 | 单一问题深入探索 |
| **广度扫描** | 新领域探索 | 多个问题并行研究 |
| **交叉验证** | 关键假设 | 多角度验证同一假设 |
| **快速迭代** | 时间敏感问题 | 快速试错 |

### 实施方案

```python
# 元规划脚本（伪代码）

def meta_plan(round_data):
    # 1. 状态分析
    progress = analyze_progress(round_data)
    
    # 2. 策略选择
    if progress['findings_quality'] > 0.7:
        strategy = 'depth_mining'
    elif progress['coverage'] < 0.5:
        strategy = 'breadth_scan'
    else:
        strategy = 'cross_validation'
    
    # 3. 计划生成
    plan = generate_plan(strategy, round_data)
    
    return plan
```
