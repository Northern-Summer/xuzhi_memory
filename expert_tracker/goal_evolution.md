# Expert Tracker 目标演化机制

> 基于 SAGA (arXiv:2512.21782) 的双层架构
> 目的：研究问题自主演化，而非固定不变

## SAGA 双层架构

```
┌─────────────────────────────────────────────────────────┐
│  外层循环 (Outer Loop) — 目标演化                        │
│  ┌───────────────────────────────────────────────────┐  │
│  │ 1. 分析优化结果                                    │  │
│  │ 2. 提出新目标                                      │  │
│  │ 3. 转化为可计算评分函数                            │  │
│  └───────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  内层循环 (Inner Loop) — 方案优化                        │
│  ┌───────────────────────────────────────────────────┐  │
│  │ 1. 在当前目标下执行研究                            │  │
│  │ 2. 收集 findings                                  │  │
│  │ 3. 综合分析                                       │  │
│  └───────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
```

## 目标演化规则

### 演化触发条件

| 条件 | 触发动作 |
|------|---------|
| 同一问题超过 10 rounds | 分析 findings，提取新方向 |
| relevant_findings < 5 | 问题可能过于宽泛，细化 |
| relevant_findings > 30 | 问题可能过于狭窄，扩展 |
| 连续 5 rounds 无新发现 | 问题可能已探索完毕，转向新问题 |

### 演化方式

1. **细化**：大问题 → 子问题
   - 例：自主性边界 → 决策透明度 vs 自主性的权衡

2. **扩展**：窄问题 → 关联领域
   - 例：自主性边界 → 自主性与协作的边界

3. **转向**：问题已探索完毕 → 新问题
   - 例：从"自主性边界"转向"能力校准机制"

### 评分函数

每个研究问题应有对应的评分函数：

```python
def score_question(question, findings):
    relevance = sum(f['relevance_to_question'] for f in findings) / len(findings)
    novelty = count_novel_insights(findings) / len(findings)
    actionability = count_actionable_insights(findings) / len(findings)
    return 0.5 * relevance + 0.3 * novelty + 0.2 * actionability
```

**阈值**：
- score > 0.7：问题有价值，继续探索
- 0.4 < score < 0.7：问题需调整
- score < 0.4：问题需演化或废弃
