## 架构设计

### 双Agent循环

```
┌─────────────────────────────────────────────────────────┐
│  PROPOSER（问题生成器）                                  │
│  ┌───────────────────────────────────────────────────┐  │
│  │ 输入：历史 findings + 系统能力评估                 │  │
│  │ 输出：新研究问题（难度自适应）                     │  │
│  │ 策略：基于 Solver 表现调整问题难度                 │  │
│  └───────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
                          ↓ 问题
┌─────────────────────────────────────────────────────────┐
│  SOLVER（问题解决器）                                    │
│  ┌───────────────────────────────────────────────────┐  │
│  │ 输入：研究问题 + 可用工具 + 时间预算               │  │
│  │ 输出：findings + 困难点 + 新假设                   │  │
│  │ 反馈：问题难度评估 + 解决质量评分                  │  │
│  └───────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
                          ↓ 反馈
                    回到 PROPOSER
```

## 实现机制

### Proposer 模块

**触发时机**：每次 Round 结束时

**问题生成策略**：

| 策略 | 描述 | 适用条件 |
|------|------|---------|
| **深化** | 从 finding 中提取子问题 | Solver 表现良好 |
| **简化** | 将复杂问题分解 | Solver 表现不佳 |
| **转向** | 提出全新方向 | 问题得分 < 0.4 |
| **关联** | 连接多个 findings | findings > 10 |

**难度自适应**：

```python
def adjust_difficulty(solver_performance):
    """
    solver_performance: 0.0-1.0
    - > 0.8: 问题太简单，提高难度
    - 0.5-0.8: 难度合适，继续深化
    - < 0.5: 问题太难，降低难度或分解
    """
    if solver_performance > 0.8:
        return "increase_difficulty"
    elif solver_performance >= 0.5:
        return "maintain_or_deepen"
    else:
        return "decrease_or_decompose"
```

### Solver 模块

**执行流程**：

```
1. 接收问题
2. 搜索相关文献
3. 提炼核心发现
4. 生成假设
5. 设计验证方案
6. 输出 findings + 困难点 + 反馈
```

**性能评估维度**：

| 维度 | 权重 | 测量方法 |
|------|------|---------|
| 文献覆盖 | 0.3 | 相关文献数 / 领域文献总数 |
| 发现质量 | 0.4 | 可迁移性 + 创新性 + 可操作性 |
| 时间效率 | 0.2 | 实际时间 / 预期时间 |
| 假设质量 | 0.1 | 可测试性 + 置信度 |

## 自动化课程学习

### 目标

> "As the solver evolves, it incentivizes the proposer to produce increasingly difficult yet solvable tasks"

**实现方式**：

1. **Round 315**：LUR 问题（难度：中等）
   - 如果 Solver 表现 > 0.8
   - Proposer 生成更难问题

2. **Round 316+**：递进问题
   - 从 LUR 基础问题 → LUR 在多Agent系统中的应用
   - 从单变量研究 → 多变量交互研究
   - 从理论模型 → 实验验证
