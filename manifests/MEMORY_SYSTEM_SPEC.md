## 10 条不可妥协原则

1. **原子性** — 写入要么完全成功，要么完全失败
2. **一致性** — 所有 agent 看到的记忆在同一时刻一致
3. **持久性** — 一旦写入，永不丢失（除非显式删除）
4. **隔离性** — 私有记忆不被意外覆盖
5. **可恢复性** — 任何故障后都能恢复到最近一致状态
6. **可追溯性** — 每条记忆都知道谁写的、何时写
7. **冲突检测** — 共享 vs 私有冲突时自动报警
8. **优先级** — Human > 共享 L1 > 私有 L1
9. **边界保护** — 大小、格式、路径全部校验
10. **同步及时性** — 写入后 5 分钟内同步到所有 agent

## 同步规则

- **方向**: 共享 L1 → 私有 L1（单向）
- **触发**: 写入后自动 + 每 5 分钟兜底
- **策略**: 原子复制 + 私有保护（私有 ≥ 共享时跳过）

## 故障恢复矩阵

| 场景 | 恢复方法 | 脚本 |
|------|---------|------|
| 文件损坏 | Git 恢复 | `git checkout` |
| 同步中断 | 重新同步 | `memory_sync.sh` |
| 私有 L1 丢失 | 从共享同步 | `memory_sync.sh` |
| Git 损坏 | 重新 clone | 手动 |
| 全系统崩溃 | 从 GitHub 恢复 | 手动 |

## 禁止行为

- ❌ 直接 write 覆盖已存在文件
- ❌ 反向同步（私有 → 共享）
- ❌ 删除日期文件
- ❌ 绕过锁机制写入
- ❌ 跳过健康检查
- ❌ 提交敏感信息到 git
