## 记忆三层架构（强制，不可更改）

```
L1 — 每日工作记忆
  位置：~/.xuzhi_memory/memory/YYYY-MM-DD.md
  定义：当天所有对话、任务、发现的原始记录
  保留期：7天 → 归档或删除
  性质：所有 agent 共享，可读写

L2 — 学科私有记忆
  位置：~/.xuzhi_memory/agents/{X}/memory/
  定义：该 agent 学科特有的知识、笔记、分析
  保留期：无期限
  性质：agent 私有，不共享，不污染 L1/L3

L3 — 永久共同记忆
  位置：~/.xuzhi_memory/memory/knowledge/
  定义：跨学科重要事实、决策、原则
  保留期：永久
  性质：所有 agent 可读
  晋升条件（必须同时满足）：
    1. 被至少2个 agent 独立验证
    2. 对系统运行有可量化影响
    3. 不依赖具体日期/事件
```

## 记忆卫生时机（强制，非可选）

### 每日晨间整理
```
检查内容：
  1. 昨日 L1 记忆 → 标记：晋升/归档/删除
  2. 检查是否有超7天未处理的 L1 文件
  3. 检查 handover 备忘录是否完整
```

### 每日暮时整理
```
执行内容：
  1. 写今日 L1 记忆摘要（3-5条最重要的）
  2. 检查今日 L2 有无需要晋升 L3 的内容
  3. 确认 handover 备忘录已写入（如有轮换）
```

### 每周一整理
```
执行内容：
  1. 遍历上周所有 L1 文件
  2. 复合评分（见下）
  3. 将高分内容晋升 L3，低分归档或删除
  4. 更新 L3 索引文件
```

### 每月末整理
```
执行内容：
  1. 遍历当月所有 L2 文件
  2. 检查学科知识是否有冲突或重复
  3. 清理无效链接、孤立文件
  4. 更新各 agent 知识索引
```

## 复合评分标准（强制计算）

```
分数 = (重要性 × 0.4) + (近期度 × 0.3) + (通用性 × 0.3)

重要性：5分=决策/原则/架构 | 3分=方法/流程/工具 | 1分=日常记录
近期度（半衰期14天）：5分=3天内 | 4分=7天内 | 3分=14天内 | 1分=30天+
通用性：5分=跨学科 | 3分=多学科 | 1分=仅本学科

评分 ≥4.0 → 考虑晋升 L3
评分 ≤1.5 → 标记删除
```

## 原子笔记标准（强制）

```
1. 一笔记一事实
   ✅ "通信开销是多智能体协作的主要瓶颈"
   ❌ "多智能体系统有很多问题……"

2. 标题即索引（可搜索）
   ✅ "# 通信开销是多智能体协作的主要瓶颈"

3. 包含链接
   每条笔记必须链接到至少一条相关笔记（[[filename]] 格式）

4. 有来源
   包含：日期、来源、验证状态
```

## 跨 agent 隔离规则（强制）

```
1. L1 是共享的，但每个 agent 只写自己的 section
2. L2 是私有的，绝对不写入他人 L2 目录
3. L3 是共享的，晋升需要理由（防止霸权）
4. handover 备忘录是 L1，但只写交出者的视角
5. 禁止：复制他人 L2 内容到自己的 L2（引用除外）
6. 禁止：未经共识修改他人在 L1 留下的内容
```

## 强制执行机制

### 每日自动检查（cron job）
```
执行：morning_memory_hygiene.py
时间：每日 07:30 Asia/Shanghai
检查：
  1. 是否有超过7天的 L1 未处理
  2. 昨日 L1 是否存在且非空
  3. handover 备忘录是否完整（周切换日）
  4. 各 agent L2 是否有交叉污染
```

### 违规处理
```
- 首次违规：警告，写入当事 agent 的 L2 + 通知评审人
- 二次违规：上报人类（微信推送）
- 系统性违规：暂停该 agent 的写权限，直至人类介入
```

### 互不监督机制（强制）
```
每次轮换日（周一 00:05），当值 agent 自动成为"评审人"：
  1. 检查上任 agent 的 handover 是否完整
  2. 检查上任 agent 是否有 L1 超时文件
  3. 检查上任 agent 的 L2 是否有污染

评审结果写入：~/.xuzhi_memory/memory/hygiene_audit_{date}.md
发现问题 → 当事 agent 的 L2 + 微信通知人类

评审链（每周循环）：
  Mon Φ 上任 → 评审 omega（上周五）
  Tue Δ 上任 → 评审 Φ
  Wed Γ 上任 → 评审 Δ
  Thu Θ 上任 → 评审 Γ
  Fri Ω 上任 → 评审 Θ
  Sat ψ 上任 → 评审 Ω
  Sun Ξ 上任 → 评审 ψ
```

## 汇报格式强制规则（所有 agent 必须遵守）

### 汇报必须包含
1. **名字**：agent 名字或 ID（不是希腊字母，是实际的 ID 如 xi/phi/delta 等）
2. **时间戳**：YYYY-MM-DDTHH:MM:SS+08:00 格式
3. **来源**：谁发的
4. **目标**：发给谁（可不写表示发给人类）

### 格式模板
```
[时间戳] sender → receiver
内容
```

### 禁止
- ❌ 不带名字的汇报
- ❌ 不带时间戳的汇报
- ❌ 匿名消息
