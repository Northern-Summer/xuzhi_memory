# 因果推断 (Causal Inference)

## 定义
因果推断是一套统计方法，用于从数据中识别变量之间的因果关系，回答"如果X发生变化，Y会如何变化"的问题。

## 核心框架

### 1. 潜在结果框架 (Rubin Causal Model)
- **基本思想**：每个个体都有潜在结果（接受处理 vs 不接受处理）
- **因果效应**：个体层面的因果效应 = Y(1) - Y(0)
- **根本问题**：无法同时观察到两个潜在结果

### 2. 因果图框架 (Pearl's Causal Graph)
- **有向无环图 (DAG)**：表示变量间的因果关系
- **后门准则**：识别需要控制的混杂变量
- **do-演算**：从观察数据推导干预效果

## 常用方法

### 随机实验
- **黄金标准**：随机分配处理，消除混杂
- **适用场景**：实验条件可控
- **局限**：成本高、伦理限制、外部效度

### 观察性研究方法

#### 1. 匹配 (Matching)
- **思想**：为处理组找到相似的对照组
- **方法**：倾向得分匹配、精确匹配
- **关键假设**：可忽略性（给定协变量，处理分配独立于结果）

#### 2. 双重差分 (Difference-in-Differences)
- **思想**：比较处理组和对照组在处理前后的变化差异
- **关键假设**：平行趋势假设
- **适用**：政策评估、自然实验

#### 3. 工具变量 (Instrumental Variables)
- **思想**：找到影响处理但直接影响结果的工具变量
- **经典例子**：安吉斯特和克鲁格的教育年限研究
- **关键假设**：相关性、排他性

#### 4. 断点回归 (Regression Discontinuity)
- **思想**：利用处理分配阈值附近的"准随机"变异
- **适用**：政策有明确资格门槛
- **优势**：因果识别强

## 操作步骤

### 1. 定义因果问题
- 明确处理变量 X
- 明确结果变量 Y
- 定义因果效应（ATE, ATT, LATE）

### 2. 识别策略
- 判断数据类型（实验/观察）
- 选择合适方法
- 验证关键假设

### 3. 估计
- 实现选定方法
- 计算因果效应
- 估计置信区间

### 4. 稳健性检验
- 检验关键假设
- 敏感性分析
- 安慰剂检验

## 质量标准
- **内部效度**：因果推断的可信度
- **外部效度**：结果可推广性
- **统计功效**：检测真实效应的能力

## 局限
- 假设难以完全验证
- 需要足够的样本量
- 复杂的统计方法容易误用
- 观察数据总有未观察到的混杂风险

## 因果机器学习（前沿整合）

### 1. 因果森林 (Causal Forest)
- **原理**：用随机森林估计异质性处理效应
- **优势**：个体层面因果效应估计
- **工具**：grf (R), EconML (Python)
- **应用**：识别谁受益最大、谁不受益

### 2. Double Machine Learning (DML)
- **原理**：ML 估计混淆变量 + 因果效应估计
- **优势**：处理高维混淆变量
- **工具**：EconML, DoubleML
- **应用**：大量协变量时的因果推断

### 3. 倾向得分 + ML
- **方法**：
  - 用 XGBoost/随机森林估计倾向得分
  - 用估计的倾向得分做匹配/加权
- **优势**：更灵活的函数形式
- **风险**：可能过拟合，需交叉验证

### 4. 因果发现 (Causal Discovery)
- **方法**：PC算法、FCI算法、LiNGAM
- **用途**：从数据中学习因果结构
- **局限**：需要强假设（如因果充分性）

## 预测-因果整合框架

### 问题：预测准确 ≠ 因果正确

**例子**：
- 预测模型：用"鞋码"预测"阅读能力"，准确率高
- 原因：混淆变量"年龄"
- 因果错误：改变鞋码不会改变阅读能力

### 整合方法

**方法1：因果特征选择**
1. 用因果图识别因果路径
2. 只用因果路径上的特征预测
3. 避免使用混淆变量

**方法2：因果正则化**
```python
# 惩罚与因果假设矛盾的模型
loss = prediction_loss + λ * causal_violation_penalty
```

**方法3：双重验证**
1. 预测性能检验
2. 因果机制检验
3. 两者必须一致
