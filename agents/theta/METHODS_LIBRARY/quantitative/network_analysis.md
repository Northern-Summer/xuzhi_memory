# 网络分析 (Network Analysis)

## 定义
网络分析是一种研究方法，将社会实体（个人、组织、概念）视为节点，将它们之间的关系视为连边，通过图论方法分析网络结构和动态。

## 适用场景
- 研究社会结构和关系
- 分析信息/资源流动
- 识别关键行动者（中心性）
- 研究网络形成和演化

## 核心概念

### 1. 网络层次
- **自我中心网络 (Ego-network)**：围绕特定节点的网络
- **整体网络 (Whole network)**：封闭系统内所有节点和连边
- **多层次网络**：个体-群体-组织跨层分析

### 2. 连边类型
- **无向**：关系对称（如合作）
- **有向**：关系不对称（如引用、关注）
- **加权**：关系强度（如互动频率）
- **多重**：多种关系并存（如朋友+同事）

### 3. 中心性指标
- **度中心性 (Degree)**：直接连接数
- **中介中心性 (Betweenness)**：控制资源流动的能力
- **接近中心性 (Closeness)**：到达所有节点的平均距离
- **特征向量中心性 (Eigenvector)**：连接高中心性节点的程度

### 4. 网络层次指标
- **密度 (Density)**：实际连边/可能连边
- **聚类系数 (Clustering)**：朋友的朋友也是朋友
- **平均路径长度**：网络连通性
- **模块化 (Modularity)**：社群结构强度

## 操作步骤

### 1. 定义网络边界
- 明确节点类型和范围
- 确定关系类型
- 设定时间范围

### 2. 数据收集
- **问卷**：提名法、名册法
- **档案数据**：合作记录、引用关系、通信记录
- **数字痕迹**：社交媒体、邮件网络

### 3. 数据清洗和构建
- 构建邻接矩阵
- 处理缺失数据
- 处理孤立节点

### 4. 网络可视化
- 布局算法（力导向、层次、环形）
- 节点大小/颜色映射属性
- 连边粗细映射权重

### 5. 网络分析
- 描述性统计
- 中心性分析
- 社群检测
- 网络比较

### 6. 统计推断（前沿）
- **指数随机图模型 (ERGM)**：网络形成机制
- **随机行动者模型 (SAOM)**：网络动态演化
- **网络回归**：控制网络自相关

## 反事实推理（关键检验）

### 检验1：边界界定的偏见
- **如果**边界定义过窄
- **那么**可能错过关键节点和连边
- **对策**：敏感性分析、多边界比较

### 检验2：关系测量的误差
- **如果**关系测量不准确（如问卷回忆偏差）
- **那么**网络结构可能失真
- **对策**：多数据源验证、稳健性检验

### 检验3：中心性的误读
- **如果**只看度中心性
- **那么**可能错过"桥接者"（高中介中心性）
- **对策**：多中心性指标综合分析

### 检验4：因果推断困难
- **如果**发现网络位置与结果相关
- **那么**可能是选择效应（相似者相连）而非因果
- **对策**：纵向设计、工具变量、实验设计

## 前沿方法（2024-2026）

### 1. 图神经网络 (GNN)
- 深度学习 + 网络结构
- 节点分类、连边预测
- 工具：PyTorch Geometric, DGL

### 2. 动态网络分析
- 时序网络演化
- 事件历史分析
- 工具：networkx-temporal, tsna

### 3. 多层网络分析
- 多类型关系整合
- 跨层影响分析
- 工具：multinetx, MuxViz

### 4. 网络因果推断
- 网络干扰下的因果效应
- 工具：netcause, interference

## 工具栈
- **Python**: networkx, igraph, graph-tool, PyTorch Geometric
- **R**: igraph, statnet, sna
- **可视化**: Gephi, Cytoscape, D3.js
- **大规模**: Neo4j, GraphX (Spark)

## 局限
- 网络边界界定主观
- 关系测量误差
- 因果推断困难
- 计算复杂度（大规模网络）
- 过度简化（二元连边忽略复杂性）

## AI辅助应用
- **网络构建**：从文本提取关系（NLP + 知识图谱）
- **连边预测**：机器学习预测缺失连边
- **社群检测**：深度学习方法
- **可视化**：自动布局优化
- **⚠️ 警告**：
  - 网络意义需研究者解释
  - 算法偏见可能影响结果
  - 数据质量决定分析上限
