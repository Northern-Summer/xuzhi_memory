# LLM 辅助研究方法论 (LLM-Assisted Research Methodology)

## 定义
LLM 辅助研究是指在大语言模型（如 GPT、Claude、DeepSeek）支持下进行社会科学研究的系统性方法，强调透明、可验证、伦理合规。

---

## 核心原则

### 1. 透明性原则 (Transparency)

**要求**：
- 报告 LLM 使用方式（提示词、模型版本、温度等）
- 区分人类贡献 vs LLM 贡献
- 公开 LLM 交互日志（涉及隐私时脱敏）

**示例声明**：
```
本文献综述使用 Claude 3.5 Sonnet (2024-10-22) 辅助文献检索
和摘要生成。所有摘要经研究者审核和修改。提示词见附录 A。
```

### 2. 可验证性原则 (Verifiability)

**要求**：
- LLM 输出必须经人类验证
- 关键论断必须有原始来源支撑
- 检测并修正"幻觉"

**验证清单**：
- [ ] 引用是否真实存在？
- [ ] 数据是否准确？
- [ ] 论证是否有逻辑漏洞？
- [ ] 是否有隐藏假设？

### 3. 可归因性原则 (Attribution)

**要求**：
- 区分"LLM 生成"、"LLM 辅助修改"、"人类原创"
- 核心理论贡献必须来自人类
- 不将 LLM 列为"作者"

**归因等级**：
| 等级 | 说明 | 声明要求 |
|------|------|---------|
| LLM 生成 | 输出直接使用 | 强制声明 |
| LLM 辅助修改 | 人类修改 >50% | 建议声明 |
| LLM 辅助检索 | 仅用于文献检索 | 可选声明 |
| 人类原创 | 无 LLM 参与 | 无需声明 |

### 4. 伦理合规原则 (Ethical Compliance)

**禁止场景**：
- ❌ 涉及人类被试的敏感研究（用 LLM 替代 IRB 审批）
- ❌ 不经审核的 LLM 输出用于政策建议
- ❌ 上传敏感数据到商业 LLM API
- ❌ 用 LLM 替代伦理审查

**允许场景**：
- ✅ 文献检索和摘要
- ✅ 代码生成和调试
- ✅ 写作润色（非核心内容）
- ✅ 数据可视化辅助

## 应用场景与方法

### 场景1：文献综述

**工作流**：
```
1. 研究者定义检索关键词
2. LLM 辅助检索（Google Scholar、arXiv API）
3. LLM 生成初步摘要
4. 研究者审核 + 验证引用
5. 研究者综合分析
6. LLM 辅助润色
```

**质量检验**：
- **幻觉检测**：每个引用必须验证 URL/DOI
- **覆盖性检验**：是否遗漏关键文献？
- **偏见检验**：LLM 是否偏好特定来源？

**提示词模板**：
```
你是一个社会科学文献综述助手。请：
1. 检索关于 [主题] 的核心文献（2020-2025）
2. 为每篇文献生成 100 字摘要
3. 提取每篇文献的核心论点和局限
4. 按主题组织文献

重要：
- 只返回真实存在的文献
- 每篇文献必须包含 DOI 或 URL
- 不确定时标注"需验证"
```

### 场景2：定性编码辅助

**工作流**：
```
1. 研究者定义编码框架
2. LLM 预编码部分数据
3. 研究者审核 + 修正
4. 计算编码者间一致性（LLM vs 人类）
5. 如一致性低，调整提示词或放弃 LLM
```

**警告**：
- LLM 不理解文化语境
- 编码定义必须由研究者提供
- 不能替代人工编码的深度理解

**一致性标准**：
- Cohen's Kappa < 0.6：LLM 不可用
- 0.6 ≤ Kappa < 0.8：可用但需大量审核
- Kappa ≥ 0.8：可用，适量审核

### 场景3：数据分析代码生成

**工作流**：
```
1. 研究者描述分析需求
2. LLM 生成代码
3. 研究者测试 + 验证
4. 在测试数据上运行
5. 验证输出正确性
```

**质量检验**：
- 在已知答案的数据上测试
- 检查代码逻辑是否正确
- 检查是否引入偏见

**提示词模板**：
```
你是一个 R/Python 数据分析助手。请：
1. 读取数据文件 [文件名]
2. 执行 [分析任务]
3. 输出结果表格

要求：
- 添加详细注释
- 检查缺失值处理
- 报告样本量
```

### 场景4：写作润色

**允许范围**：
- 语法修正
- 表达优化
- 格式规范

**禁止范围**：
- 生成核心论证
- 填充文献引用
- 创造"新观点"

**工作流**：
```
1. 研究者完成初稿
2. LLM 润色语言
3. 研究者审核修改
4. 恢复被错误修改的内容
```

## 幻觉检测与预防

### 幻觉类型

| 类型 | 表现 | 风险等级 |
|------|------|---------|
| 虚构引用 | 不存在的论文 | 🔴 高 |
| 数据捏造 | 编造统计数据 | 🔴 高 |
| 逻辑跳跃 | 无前提的结论 | 🟡 中 |
| 过度概括 | 从个案到普遍 | 🟡 中 |
| 概念混淆 | 错误定义术语 | 🟡 中 |

### 检测方法

**方法1：引用验证**
```python
def verify_citation(title, authors, year):
    # 用 DOI API 或 Google Scholar 验证
    result = search_scholar(title)
    return result.authors == authors and result.year == year
```

**方法2：数据溯源**
- 要求 LLM 提供数据来源
- 追溯到原始数据集
- 验证统计量是否匹配

**方法3：逻辑审查**
- 要求 LLM 展示推理步骤
- 检查每一步的前提
- 识别跳跃性推理

**方法4：交叉验证**
- 用不同 LLM（如 GPT + Claude）生成相同内容
- 对比差异
- 差异大 → 可能是幻觉

### 预防策略

**提示词工程**：
```
重要：你必须只使用真实存在的信息。
如果不确定，请明确说明"需要验证"。
不要编造引用、数据或事实。
```

**温度设置**：
- 事实性任务：temperature = 0
- 创造性任务：temperature > 0.5（但需验证）

**引用约束**：
- 要求 LLM 只引用已知来源
- 提供可信来源列表

## 偏见检测与缓解

### LLM 偏见来源

| 来源 | 表现 | 缓解方法 |
|------|------|---------|
| 训练数据 | 文化/语言偏见 | 使用领域微调模型 |
| 对齐偏见 | RLHF 引入的偏见 | 对比多个模型 |
| 提示词偏见 | 引导性提问 | 盲审提示词 |
| 语境偏见 | 前文影响后文 | 分段处理 |

### 检测方法

**方法1：对比测试**
- 相同任务，不同群体描述
- 检查输出差异

**方法2：反向测试**
- 反转假设，检查输出是否对称

**方法3：多样性审计**
- 检查 LLM 输出的多样性
- 是否系统性忽略某些群体

### 缓解策略

1. **多样化输入**：提供多种视角的输入
2. **对比多个 LLM**：不依赖单一模型
3. **人类审核**：关键决策必须人工
4. **偏见标注**：记录已知偏见

## 数据隐私与安全

### 禁止上传的数据

- ❌ 个人身份信息（PII）
- ❌ 医疗健康数据
- ❌ 未脱敏的访谈记录
- ❌ 受限研究数据（如保密协议）
- ❌ 商业敏感信息

### 安全实践

**本地部署优先**：
- 使用本地 LLM（如 Ollama、vLLM）
- 避免上传敏感数据到 API

**脱敏处理**：
```python
def anonymize_text(text):
    # 移除姓名
    text = remove_names(text)
    # 移除地址
    text = remove_addresses(text)
    # 移除 ID 号
    text = remove_ids(text)
    return text
```

**API 使用规范**：
- 使用企业版 API（数据不用于训练）
- 审阅 API 服务条款
- 避免上传原始数据

## 学术伦理

### 期刊政策（2024-2026 主流立场）

| 期刊 | 政策 | 要求 |
|------|------|------|
| Nature | 允许使用 | 必须在方法中声明 |
| Science | 允许使用 | 必须声明 + 人负责 |
| ASR | 允许使用 | 必须声明 + 可验证 |
| AJS | 允许使用 | 核心贡献必须人类 |

### 推荐声明模板

```
本文使用大语言模型 [模型名称] 辅助 [具体用途]。
研究者对本文所有内容负责。LLM 使用的详细信息见附录。
```

### 禁止行为

- ❌ 将 LLM 列为作者
- ❌ 不声明 LLM 使用
- ❌ 不验证 LLM 输出
- ❌ 用 LLM 替代 IRB 审批

## 质量清单

### 使用前
- [ ] 是否有明确的使用目的？
- [ ] 是否选择了合适的模型？
- [ ] 是否设计了清晰的提示词？
