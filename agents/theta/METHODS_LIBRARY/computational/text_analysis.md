# 文本分析 (Text Analysis)

## 定义
文本分析是一系列方法，用于从文本数据中提取有意义的信息、模式和洞见。

## 适用场景
- 大规模文献分析
- 社交媒体研究
- 档案和史学研究
- 新闻和媒体分析

## 方法谱系

### 1. 词典方法 (Dictionary Methods)
- **原理**：预定义词汇表计数
- **优点**：可解释、无需训练
- **缺点**：依赖词典质量、忽略上下文
- **应用**：情感分析、政策不确定性测量

### 2. 主题模型 (Topic Modeling)
| 方法 | 特点 |
|------|------|
| LDA | 生成式概率模型 |
| NMF | 矩阵分解 |
| STM | 结构主题模型（含协变量） |
| BERTopic | 嵌入 + 聚类 |

### 3. 词嵌入方法
| 方法 | 特点 |
|------|------|
| Word2Vec | 静态词嵌入 |
| GloVe | 全局共现矩阵 |
| BERT embeddings | 上下文嵌入 |
| 历史嵌入 | 追踪语义演变 |

### 4. 监督学习
- 文本分类
- 情感分析
- 命名实体识别

### 5. 定性-定量混合
- 计算辅助定性分析
- 人机协同编码
- 解释性文本挖掘

## 操作步骤

### 1. 数据收集
- **社交媒体**: Twitter API, 微信公众号
- **新闻**: GDELT, Factiva
- **档案**: HathiTrust, Google Books
- **学术文献**: Web of Science, Scopus

### 2. 数据预处理
- 清洗和标准化
- 分词（中英文处理不同）
- 去停用词（谨慎）
- 词形还原/词干提取

### 3. 探索性分析
- 词频统计
- 词云可视化
- 共现网络

### 4. 模型应用
- 选择适当方法
- 调优参数
- 验证结果

### 5. 解释和验证
- 主题命名（研究者任务）
- 与理论对话
- 定性验证

## 反事实推理（关键检验）

### 检验1：词典适用性
- **如果**使用通用情感词典
- **那么**可能不适用于特定领域
- **对策**：领域词典、验证词典效度

### 检验2：主题数量
- **如果**主题数选择不当
- **那么**主题可能过宽或过窄
- **对策**：多主题数比较、一致性指标、定性评估

### 检验3：语义漂移
- **如果**分析跨越长时间段
- **那么**同一词的含义可能变化
- **对策**：历史嵌入、分期分析

### 检验4：文本代表性
- **如果**文本来源有偏
- **那么**结论可能不具代表性
- **对策**：多来源验证、偏差校正

## 主题模型验证：四层框架（关键补充）

### Layer 1: 统计有效性
- **一致性指标**：C_v, U_mass
- **困惑度**：Perplexity（但已证明不可靠）
- **问题**：统计有效 ≠ 语义有效

### Layer 2: 语义有效性
- **研究者审核**：主题词是否连贯？
- **主题命名**：能否用一句话概括？
- **对齐检验**：主题是否与研究问题对齐？

### Layer 3: 预测有效性
- **外部变量预测**：主题能否预测已知变量？
- **时间趋势预测**：主题流行度是否合理？
- **交叉验证**：训练集主题在测试集是否稳定？

### Layer 4: 理论有效性
- **理论对话**：主题是否与理论框架一致？
- **异常解释**：异常主题是否有理论意义？
- **新假设生成**：主题是否启发新研究问题？

### Grimmer 验证清单（必执行）
- [ ] 报告多种主题数的结果（如 K=5,10,15,20）
- [ ] 每个主题命名需研究者完成
- [ ] 定性验证关键文档（抽样阅读）
- [ ] 比较不同模型（LDA vs STM vs BERTopic）
- [ ] 报告主题稳定性（多次运行结果）

## 前沿方法（2024-2026）

### 1. LLM 驱动的文本分析
- **零样本分类**：无需训练数据
- **主题提取**：LLM 生成主题标签
- **摘要和提取**：自动生成摘要
- **对话式分析**：与数据交互

### 2. 结构主题模型 (STM)
- 协变量影响主题流行度
- 内容相关协变量
- 工具：R stm 包

### 3. 动态主题模型
- 追踪主题随时间演化
- 识别新兴话题
- 工具：DTM, Dynamic TM

### 4. 跨语言文本分析
- 多语言嵌入 (LASER, LaBSE)
- 零样本跨语言迁移
- 比较研究

### 5. 文本因果推断
- 文本作为处理变量
- 文本作为混淆变量
- 文本作为结果变量
- 工具：CausalText

## 工具栈
- **Python**:
  - 基础: NLTK, spaCy
  - 主题模型: gensim, tomotopy
  - 嵌入: sentence-transformers
  - LLM: langchain, transformers
- **R**:
  - 主题模型: stm, topicmodels
  - 文本分析: tidytext, quanteda
- **可视化**:
  - pyLDAvis（主题可视化）
  - wordcloud
  - UMAP/t-SNE（嵌入可视化）

## 局限
- 文本 ≠ 现实（话语 vs 实践）
- 上下文丢失（脱离语境）
- 数据偏差（谁在说话？）
- 算法黑箱
- 过度依赖技术，忽视理论

## AI辅助应用（元认知）

### LLM 用于文本分析
- **自动编码**：LLM 辅助定性编码
- **主题生成**：自动生成主题标签
- **验证辅助**：识别异常模式
- **报告生成**：自动撰写分析报告

### ⚠️ 警告
- LLM 可能过度解读
- 研究者必须理解算法原理
- 理论指导是核心，技术是手段
