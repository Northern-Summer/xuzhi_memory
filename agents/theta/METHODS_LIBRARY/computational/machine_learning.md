# 机器学习 (Machine Learning)

## 定义
机器学习是一类算法，通过数据自动学习模式，进行预测或决策，无需显式编程规则。

## 适用场景
- 预测建模
- 分类和聚类
- 模式识别
- 自动化决策

## 核心范式

### 1. 监督学习 (Supervised Learning)
- **任务**：从标注数据学习映射函数
- **方法**：
  - 分类：逻辑回归、SVM、决策树、神经网络
  - 回归：线性回归、随机森林、梯度提升
- **评估**：准确率、F1、AUC、MSE

### 2. 无监督学习 (Unsupervised Learning)
- **任务**：发现数据内在结构
- **方法**：
  - 聚类：K-means、层次聚类、DBSCAN
  - 降维：PCA、t-SNE、UMAP
  - 主题模型：LDA、NMF
- **评估**：轮廓系数、重构误差

### 3. 强化学习 (Reinforcement Learning)
- **任务**：智能体在环境中学习最优策略
- **方法**：Q-learning、Policy Gradient、PPO
- **应用**：社会模拟、决策优化

### 4. 深度学习 (Deep Learning)
- **架构**：CNN、RNN、Transformer、GNN
- **优势**：自动特征提取、处理复杂模式
- **局限**：黑箱、需要大量数据

## 操作步骤

### 1. 问题定义
- 明确任务类型（分类/回归/聚类）
- 定义目标变量和特征
- 确定评估指标

### 2. 数据准备
- 数据清洗
- 特征工程
- 划分训练/验证/测试集
- 处理不平衡数据（SMOTE、类权重）

### 3. 模型选择
- 从简单模型开始（基线）
- 逐步尝试复杂模型
- 考虑可解释性需求

### 4. 模型训练
- 超参数调优（Grid Search, Random Search, Bayesian Optimization）
- 交叉验证
- 正则化防止过拟合

### 5. 模型评估
- 测试集性能
- 混淆矩阵分析
- 错误案例分析

### 6. 解释和部署
- 特征重要性
- SHAP/ LIME 解释
- 监控模型漂移

## 反事实推理（关键检验）

### 检验1：数据泄露
- **如果**训练数据包含测试信息
- **那么**性能被高估
- **对策**：严格的时间划分、特征检查

### 检验2：过拟合
- **如果**模型过于复杂
- **那么**训练集表现好，测试集差
- **对策**：正则化、早停、简化模型

### 检验3：特征重要性误读
- **如果**特征相关
- **那么**重要性可能被重新分配
- **对策**：置换重要性、SHAP

### 检验4：群体差异
- **如果**训练和测试群体不同
- **那么**模型可能对某些群体不公平
- **对策**：群体公平性评估、对抗训练

## 前沿方法（2024-2026）

### 1. 可解释 AI (XAI)
- **全局解释**：特征重要性、部分依赖图
- **局部解释**：SHAP、LIME、Counterfactual explanations
- **模型无关**：适用于任何黑箱模型

### 2. 因果机器学习
- **因果森林**：异质性处理效应估计
- **Double Machine Learning**：因果推断 + ML
- **工具**：EconML, CausalML

### 3. 公平机器学习
- **公平性指标**：Demographic Parity, Equalized Odds
- **去偏见方法**：预处理、中处理、后处理
- **工具**：fairlearn, AIF360

### 4. 自动化机器学习 (AutoML)
- 自动特征工程
- 自动模型选择
- 工具：AutoGluon, TPOT, H2O

### 5. 联邦学习
- 数据不出本地，协作训练
- 隐私保护
- 应用：多机构合作研究

## 泛化检验（关键补充）

### 检验1：样本选择偏见
**问题**：训练样本可能不具代表性

**检验方法**：
- 比较训练样本与目标群体的分布
- 使用加权或重采样校正
- 报告样本选择标准

**反事实检验**：
- 如果换一个样本，模型性能如何变化？

### 检验2：情境依赖
**问题**：模型可能过拟合特定情境

**检验方法**：
- 跨时间测试（训练旧数据，测试新数据）
- 跨空间测试（训练地区A，测试地区B）
- 跨平台测试（训练平台X，测试平台Y）

**反事实检验**：
- 如果换一个文化/时间/平台，结论是否成立？

### 检验3：模型过拟合
**问题**：模型可能记住训练数据的特殊模式

**检验方法**：
- 学习曲线分析（性能 vs 样本量）
- 嵌套交叉验证
- 置换检验

**反事实检验**：
- 模型是否学到真实的规律，还是噪音？

### 检验4：概念漂移
**问题**：概念含义可能随时间变化

**检验方法**：
- 监控模型性能衰减
- 定期重训练
- 检测特征分布变化

**反事实检验**：
- 如果概念含义变化，模型是否仍然有效？

## Yarkoni 泛化危机应对（2020）

**核心洞见**：
> "当前研究过度依赖特定数据集，结论不可泛化。"

**应对策略**：

### 策略1：科学后悔最小化（Agrawal et al. 2020）
- 设计实验时考虑多种可能的结论
- 减少对特定结果的依赖
- 最大化学习的普适性

### 策略2：多研究序列设计
- 第1研究：探索性
- 第2研究：预注册验证
- 第3研究：跨样本复制

### 策略3：透明报告不确定性
- 报告置信区间，不只点估计
- 讨论边界条件
- 承认泛化局限

## 局限
- 数据需求大
- 黑箱问题
- 外推风险
- 偏见放大
- 因果推断困难
- 过度依赖技术，忽视理论
- **泛化危机**：特定数据集结论不可泛化

## AI辅助应用（元认知）

### LLM 用于 ML 研究
- **代码生成**：自动生成分析代码
- **超参数建议**：基于经验的推荐
- **结果解释**：生成分析报告
- **文献检索**：查找相关方法

### ⚠️ 警告
- ML 只是工具，理论是核心
- 不加思考地应用 ML = 数据挖掘
- 因果推断需要识别策略，非纯预测
