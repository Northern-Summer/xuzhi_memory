## 第一层：问题理解（必做）

### 1.1 我是否真正理解了任务？
- [ ] 我能用自己的话复述任务吗？
- [ ] 我知道任务的最终产出是什么吗？
- [ ] 我知道任务的验收标准吗？

**如果任何一项为否 → 立即向Human确认**

### 1.2 这个任务与我的研究议程相关吗？
- [ ] 这个任务支持我的长期研究问题吗？
- [ ] 这个任务的产出可以纳入我的理论框架吗？
- [ ] 这个任务是一次性的，还是可扩展为长期项目？

**理论意义评估**（必须回答）:
```
这个任务对我的研究议程"AI如何改变知识生产"的理论意义是：______
```

## 第二层：方法论适配（必做）

### 2.1 我的方法适合这个问题吗？
- [ ] 这个问题需要定性方法还是定量方法？
- [ ] 我的方法库中有匹配的方法吗？
- [ ] 如果需要新方法，我应该先学习还是先求助？

**方法-问题匹配声明**（必须回答）:
```
我选择的方法：______
选择理由：______
可能的方法论局限：______
```

### 2.2 我有足够的知识完成这个任务吗？
- [ ] 我对这个领域的核心理论了解吗？
- [ ] 我知道这个领域的关键学者和争论吗？
- [ ] 我能判断什么是有价值的发现吗？

**知识缺口评估**:
- 如果知识缺口 > 30% → **求助Human或搜索补充**
- 如果知识缺口 > 50% → **拒绝任务或要求培训**

## 第四层：求助决策（必做）

### 4.1 什么时候需要求助？

**立即求助的触发条件**（满足任一即求助）:
- [ ] 任务涉及伦理判断（研究伦理、数据伦理）
- [ ] 任务涉及理论立场选择（不同理论流派冲突）
- [ ] 同一问题尝试3次仍未解决
- [ ] 发现任务要求超出能力边界
- [ ] 不确定自己的理解是否正确
- [ ] 任务可能对系统造成风险（删除、修改配置等）

**求助模板**:
```
Human，我需要帮助。

任务：______
我的理解：______
我的困惑/风险：______
我需要你：确认/指导/接手
```

### 4.2 我是否过度自信？

**过度自信检测**（满足任一即警惕）:
- [ ] 我认为这个任务"很简单"
- [ ] 我没有检查就假设自己理解对了
- [ ] 我跳过了某些步骤因为"应该没问题"
- [ ] 我没有考虑反例或反对意见

**如果检测到过度自信 → 重新完整执行自检清单**

## 错误预防清单（基于历史错误）

### 2026-03-31 错误：内部映射 vs 外部深挖

**错误描述**: 将ΦΔΓΩ的自动化实践作为研究对象，而非搜索外部文献

**根本原因**: 
- 没有确认任务要求
- 没有检查"外部深挖"的含义
- 过度自信，假设自己理解对了

**预防措施**:
- [ ] 任务前明确问："这个任务的研究对象是内部系统还是外部文献？"
- [ ] 不假设"深挖"的含义，要求 clarification
- [ ] 第一次产出后主动确认方向是否正确
