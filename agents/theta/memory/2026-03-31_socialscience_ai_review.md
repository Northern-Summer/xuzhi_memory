## 一、社会科学领域

### 1.1 scholar-skill: 端到端研究智能体

**来源**: Yongjun Zhang (2026) *Vibe Researching as Wolf Coming: Can AI Agents with Skills Replace or Augment Social Scientists?* arXiv:2602.22401

**核心贡献**:
- **26个技能插件**覆盖从研究构思到投稿的完整流程
- **18个编排阶段** + **53个质量关卡**的结构化工作流
- **认知任务框架**: 按可编码性(codifiability)与隐性知识要求(tacit knowledge)划分人机边界

**方法论内核**:
```
人机边界划分:
- 高可编码+低隐性 → AI承担 (文献检索、代码生成、格式规范)
- 低可编码+高隐性 → Human保留 (理论原创、语境判断)

阶段-关卡结构:
感知→识别→处理→验证 (每个阶段设自动检查项)

技能原子化:
每个技能独立可调用、明确输入输出、可组合替换
```

**可复用模式**:
1. 建立"阶段-关卡"清单，AI逐阶段推进
2. 复杂任务拆分为最小技能单元
3. 在流程中明确标注"必须由人类决策"的节点

---

### 1.2 SCALE: 多智能体协作内容分析

**来源**: Zhao et al. (2025) *SCALE: Towards Collaborative Content Analysis in Social Science with Large Language Model Agents and Human Intervention.* ACL 2025 Main Conference. arXiv:2502.10937

**核心贡献**:
- **多智能体模拟**内容分析三阶段: 文本编码→协作讨论→代码簿演化
- **动态代码簿**: 智能体在讨论中可提议修改分类体系
- **三种人类干预模式**: 轻度(仅审核)/中度(参与讨论)/重度(直接修改)

**方法论内核**:
```
角色分工: 多个智能体扮演独立编码员→独立编码→讨论达成共识

动态演化: 代码簿(分类体系)在过程中可动态调整

人机交互协议:
- 每次输出包含: 人类决策点、不确定性声明、证据溯源
```

**可复用模式**:
1. 部署多个智能体扮演独立编码员，提高编码一致性
2. 让智能体自主演化分类体系，人类做最终审核
3. 设计可插拔的人类干预点，按需选择介入深度

---

### 1.3 CMASE: 计算多智能体社会实验

**来源**: Zhang et al. (2025) *Computational Multi-Agents Society Experiments: Social Modeling Framework Based on Generative Agents.* arXiv:2508.17366

**核心贡献**:
- **生成式智能体** + **虚拟民族志方法**
- 研究者从"外部操作者"变为"嵌入式参与者"
- 支持实时人机交互、机制导向干预

**方法论内核**:
```
生成式智能体: 每个智能体有身份、记忆、目标、社交网络

虚拟民族志: 研究者"进入"模拟世界，与智能体对话、观察行动

因果推断嵌入: 通过控制模拟参数隔离因果关系
```

**可复用模式**:
1. 构建生成式智能体社会，运行"如果...会怎样"对照实验
2. 将研究者设计为嵌入式参与观察者
3. 结合计算实验的严谨性与民族志的解释深度

### 1.4 AgentSociety: 大规模社会模拟

**来源**: Piao et al. (2025) *AgentSociety: Large-Scale Simulation of LLM-Driven Generative Agents.* arXiv:2502.08691

**核心贡献**:
- 整合LLM驱动智能体、真实社会环境、大规模模拟引擎
- 捕捉人类行为及其背后的机制
- 为社会科学研究和政策制定提供平台

### 1.5 台湾人文社会科学AI协作工作流

**来源**: Huang (2026) *From Labor to Collaboration: A Methodological Experiment Using AI Agents to Augment Research Perspectives in Taiwan's Humanities and Social Sciences.* arXiv:2602.17221

**核心贡献**:
- **七阶段模块化工作流**
- 基于三大原则: 任务模块化、人机分工、可验证性
- 提出三种人机协作模式: 直接执行/迭代优化/人类主导

**方法论内核**:
```
七阶段工作流:
1. 研究问题形成
2. 文献检索与综述
3. 理论框架构建
4. 研究设计
5. 数据收集与分析
6. 结果解释
7. 报告撰写与伦理审查

人机分工原则:
- Human: 研究判断、伦理决策、理论解释
- AI: 信息检索、文本生成、数据处理
```

## 二、历史学/数字人文领域

### 2.1 DeepPast: 模块化档案解读框架

**来源**: Javier Cha (2026) *Automating the Past: Artificial Intelligence and the Next Frontiers of Digital History.* International Journal of Humanities and Arts Computing (IJHAC).

**核心贡献**:
- **模块化AI架构**: 知识库与语言处理分离
- **低功耗优先**: 可在普通笔记本运行，拒绝超大模型
- **结构化对话**: AI与历史学家多轮问答

**方法论内核**:
```
模块化设计: 即插即用组件(文本预处理、语义嵌入、视觉分析、时间线对齐)

低功耗优先: 刻意避免使用超大模型，确保可持续、可本地部署

结构化对话:
- 不是一次性输出结果
- 多轮问答: "这段文本的情感倾向是什么？"
- 追问: "你识别出哪些证据支持这个判断？"
```

**可复用模式**:
1. 为历史材料设计模块化分析流水线
2. 与AI建立结构化对话，让它输出结论+证据+推理路径
3. 优先选择可本地运行的低功耗模型

### 2.2 UVic RAG+VLM: 历史报纸OCR处理

**来源**: University of Victoria Libraries (2026) *Unlocking the Past: GenAI in Library Archival Research.* (EurekAlert!科学新闻)

**核心贡献**:
- 解决历史报纸OCR质量极差的问题
- **视觉语言模型(VLM)**"看懂"版面布局(破损、折痕、歪斜)
- **检索增强生成(RAG)**确保输出基于权威档案

**方法论内核**:
```
VLM版面理解: 整体识别版面元素(标题、正文、图片、广告)，理解阅读顺序

RAG事实保障: 将识别出的文本块与权威档案库比对，用检索结果修正识别错误

人机协同校正: 系统输出初步转录后，历史学家抽查关键段落
```

**可复用模式**:
1. 材料质量差(破损、字体怪异)时用VLM代替传统OCR
2. 用RAG将提取结果与权威数据库比对，纠正错误
3. 设计主动学习循环: 人类只抽查最关键部分

### 2.3 LLM历史文档OCR评估框架

**来源**: Levchenko et al. (2025) *Evaluating LLMs for Historical Document OCR: A Methodological Framework for Digital Humanities.* arXiv:2510.06743. LM4DH 2025 Workshop.

**核心贡献**:
- 提出历史OCR专用评估指标
- **HCPR** (Historical Character Preservation Rate)
- **AIR** (Archaic Insertion Rate)
- 发现过度历史化问题: LLM会插入错误历史时期的古字符

**方法论内核**:
```
专用评估指标: 传统指标无法捕捉时间偏差和时期特定错误

污染控制协议: 解决训练数据污染问题

稳定性测试: 确保模型输出一致性
```

## 三、通用方法论框架

### 3.1 认知任务框架 (Cognitive Task Framework)

**来源**: scholar-skill (Zhang 2026)

| 维度 | 高可编码+低隐性 | 低可编码+高隐性 |
|------|----------------|----------------|
| **AI承担** | 文献检索、代码生成、格式规范、数据清洗 | - |
| **Human保留** | - | 理论原创、语境判断、伦理决策 |

**关键洞察**: 人机边界是**认知的**，不是**序列的**——它贯穿研究流程的每个阶段，而非阶段之间。

---

### 3.2 阶段-关卡结构 (Phase-Gate Structure)

**来源**: scholar-skill + SCALE

```
通用模板:
1. 感知/扫描 → 2. 识别/编码 → 3. 处理/讨论 → 4. 验证/审查

每个阶段设置质量关卡:
- 数据来源是否可追溯？
- 分析代码是否可复现？
- 结论是否有替代解释？
- 是否存在偏见或伦理风险？
```

---

### 3.3 技能原子化 (Skill Atomicity)

**来源**: scholar-skill + DeepPast

**原则**:
1. 每个技能独立可调用
2. 明确的输入输出规范
3. 可组合、可替换、可调试

**示例技能**:
```python
# 文献检索技能
输入: 研究主题、关键词列表、数据库列表
输出: 相关文献列表(含摘要、引用数、可获取性)

# 代码生成技能
输入: 数据处理需求、数据格式描述
输出: 可执行代码(含注释、错误处理、测试用例)

# 质量审查技能
输入: 分析结果、原始数据
输出: 审查报告(可追溯性、可复现性、偏见评估)
```

---

### 3.4 人机交互协议

**来源**: SCALE + 台湾研究

**AI每次输出必须包含**:
1. **人类决策点**: 明确哪些选择需要人类介入
2. **不确定性声明**: 估计结论的可信度(如"中等证据强度")
3. **证据溯源**: 每项主张对应的数据/文献/分析结果

## 四、可复用的实施框架

### 4.1 任务拆解模板

```
任何研究任务拆成5个原子步骤:
1. 检索/采集 → 2. 清洗/结构化 → 3. 分析/建模 → 4. 解释/论证 → 5. 质量审查

每个步骤内部再细分(如"分析"拆为: 特征工程→模型选择→参数调优→稳健性检验)
```

### 4.2 学科特化技术栈

**社会科学**:
- 因果推断库: DoWhy, CausalML
- 多智能体模拟: Mesa + LLM
- 文本分析: 主题模型 + 情感分析 + 框架识别

**历史学/数字人文**:
- VLM OCR: TrOCR + LayoutLMv3
- 实体链接: 与历史知识图谱对齐
- 跨语言转写: mT5微调

## 六、研究局限与未来方向

### 6.1 当前局限
1. **数据污染风险**: LLM训练数据可能包含待分析文本
2. **过度历史化**: LLM可能生成看似合理但实际错误的历史细节
3. **平台依赖**: 多数案例依赖特定AI平台(Claude, GPT-4)
4. **评估标准缺失**: 缺乏跨学科统一的AI辅助研究评估框架

### 6.2 未来方向
1. **跨学科方法整合**: 将计算社会科学与数字人文方法融合
2. **伦理框架完善**: 建立AI辅助历史/社会科学研究的伦理审查标准
3. **开源工具生态**: 开发可本地部署的开源替代方案
4. **长期影响评估**: 追踪AI工具对研究质量和知识生产的长期影响
