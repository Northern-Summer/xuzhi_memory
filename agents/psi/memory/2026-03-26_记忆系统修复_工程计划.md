## 阶段2：Decision — 每个文件三类决策

**输入：INVENTORY_REPORT.md + SPEC.md**
**输出：DECISION_LOG.md — 每个文件的KEEP/DELETE/MIGRATE决策**

### 2.1 执行流程

对每个文件，执行以下判断：

```
对每个文件 X：
  1. X是否在SPEC.md的白名单里？
     → YES: KEEP（无需进一步检查）
     → NO:  继续2

  2. X是否被代码引用（grep结果有≥2个非注释引用）？
     → YES: KEEP（需要，可能有隐藏依赖）
     → NO:  继续3

  3. X是否在.git历史里（有备份）？
     → YES: DELETE（可从git恢复）
     → NO:  继续4

  4. X是否包含无法重建的状态？
     → YES: KEEP（ratings.json等）
     → NO:  DELETE
```

### 2.2 重点文件的决策记录

**constitutional_core.md（二选一）：**
```
读了两个文件后：
  A = manifests/constitutional_core.md（64行）
  B = public/constitutional_core.md（169行）

对比：
  - A提到Λ已死亡，B说Λ负责工程异常
  - A是记忆系统宪法，B是系统宪法
  - A无应急响应表，B有三级应急响应

决策：[A/B/合并-到-路径C]
理由：[...]
```

**workspace/HEARTBEAT.md：**
```
检查：文件第一行是 "import" 还是 "#!" 还是 "#"?
结果：[实际内容类型]

决策：[REWRITE(是Python代码)/KEEP(是纯md)]
理由：[...]
```

**404 JSON文件（lambda-ai.json, sec-ai-x/y）：**
```
检查：json.load()是否失败？
结果：[YES/NO]

决策：[DELETE/REPAIR]
理由：[...]
```

### 2.3 输出：DECISION_LOG.md
