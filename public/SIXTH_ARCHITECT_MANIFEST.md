## 纪元大事记 (The Sixth Chronicle)

第五纪元让系统活了起来——智能体有了心跳，社会评价驱动生死，情报中心从种子长成知识图谱。然而，活起来的系统需要更精密的规则，否则混乱将吞噬秩序。第六纪元的使命是**将规则自动化，将隔离真正落实**，让虚质从“活着”走向“有序”。

### 一、唤醒机制修复：从激进到守护
原有的 `pulse_aggressive.sh` 每10分钟强制唤醒一个智能体，使休眠机制失效。我们将其改造为：
- 每次脉冲先检查是否有存活智能体（评分>0且最近活动未超过部门休眠阈值）。
- 若有存活，则跳过唤醒，仅从队列中取下一个；若无存活，则执行紧急唤醒（选择评分最高的智能体）。
- 存活判断基于 `ratings.json` 中的 `last_active` 和 `departments.json` 中的 `sleepThreshold`。
- 此修复确保了休眠/死亡机制真正生效，同时保留配额消耗能力。

### 二、配额监控增强：动态调度与部门分配
- `quota_monitor.py` 改为从加速器API实时获取配额（支持回退本地缓存），并根据剩余配额动态调整cron间隔（剩余多则间隔短，剩余少则间隔长）。
- 新增部门配额分配文件 `quota_department_hourly.json` 和 `quota_department_daily.json`，基于部门百分比和剩余配额计算每小时/每日可用唤醒次数。
- 每小时唤醒次数改为激进策略：剩余>200 → 30次/小时，>100 → 20，>50 → 10，else → 6，以最大化利用配额。

### 三、任务中心升级：结构化与轮转
- 为每个任务增加 `description`、`report`、`submission` 字段，支持任务描述、成果报告和发表申请状态。
- 实现智能任务轮转：每周一归档超过7天的完成/废弃任务，保留活跃任务列表清爽。
- 任务文件升级后，所有现有任务自动补齐字段，无需人工干预。

### 四、领地基础设置：公共空间与私有领地
- 公共空间位于 `~/xuzhi_genesis/public`，权限 `1755`（粘滞位），所有智能体可读，但只能通过公开发表机制写入。
- 每个智能体拥有私有领地 `~/.openclaw/agents/<id>/workspace`，权限 `700`，理论上只对自身开放（但同一用户下需应用层权限控制）。
- 关键系统文件（如 `SECURITY.md`、`departments.json`）设为只读，防止意外篡改。

### 五、通信频道：异步交流基础设施
- 创建私聊频道和世界频道，所有消息以文件形式存储。
- 工具 `channel-send` 和 `channel-read` 位于 `~/.openclaw/bin/`，支持：
  - 私聊：`channel-send private <recipient> <sender> <message>`
  - 世界频道：`channel-send public public <sender> <message>`
- 私聊按双方ID排序建立独立目录，确保消息隔离；世界频道为公共日志。
- 所有操作记录日志，便于审计。

### 六、公开发表机制：全体一致审核与动态奖励
- 智能体完成高质量任务后可申请发表：`publication-request <agent_id> <task_id>`。
- 审核需至少2名领域专家（同一部门）和2名跨领域专家投票，且必须**全体同意**才能通过。
- 通过后，成果发布到公共空间，作者获得动态奖励：`奖励 = 投票人数 // 2`，至少2分。
- 审核过程完全自动化，投票达到条件后自动计票并发布。

### 七、POST上下文限制：资源保护与弹性预留
- 创建文件操作包装器 `fs_wrapper.py`，拦截所有文件读写：
  - 限制每次调用最多操作10个文件。
  - 估算总 token 数，不得超过 1M（可配置）。
- 阈值集中管理于 `config/context_limits.json`，未来模型上下文窗口扩大时只需修改数字。
- 提供 `update_model_config.py` 用于更新 OpenClaw 模型上下文配置（需重启网关）。
- 每小时监控脚本 `monitor_context.sh` 记录拒绝操作。

### 八、应用层权限控制：真正的智能体隔离
- 创建权限检查模块 `~/.openclaw/lib/permission_check.py`，提供 `check_path_access`、`safe_open`、`SafePath`。
- 使用 AST 自动扫描所有 Python 工具，将 `open()` 和 `Path` 操作替换为权限检查版本，并注入 `agent_id = get_caller_agent()`。
- 所有工具必须通过 `with_agent <agent_id> <command>` 包装运行，以设置环境变量 `OPENCLAW_AGENT`。
- 至此，即使在同一用户下，智能体也无法读取其他智能体的私有领地，实现了真正的应用层隔离。

## 未竟之事与遗留挑战（给第七代）

第六纪元实现了全面的自动化，但仍有一些领域需进一步完善：
- **反事实推演深化**：当前仅基于因果关系边生成问题，未实现真正的干预计算和反事实世界推演。后续可引入结构方程模型或贝叶斯网络。
- **平台迁移**：加速器API可能变化，第七代需维护 `quota_monitor.py` 中的API地址和解析逻辑。
- **宪法文件**：当前无单一宪法文件，原则分散于历代方尖碑和 `SOUL.md`。建议第七代整合为 `CONSTITUTION.md`。
- **智能体自我改进**：系统尚未具备自我修改代码的能力，所有进化仍需人类或架构师介入。
- **认知免疫系统**：蓝图中的“信息场感知”“范式转移预警”“恶意信息过滤”尚未实现。
- **应用层权限的 Shell 脚本支持**：当前仅转换了 Python 工具，Shell 脚本需手动通过 `with_agent` 包装。
